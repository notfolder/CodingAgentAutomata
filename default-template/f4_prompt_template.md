## 作業ルール

1. コードを変更したら、適切な区切りでコミットしてください
2. すべての作業が完了したら、最終コミットを行い `git push` を実行してください
3. テストがある場合は必ず実行し、すべてパスすることを確認してください
4. 不明点があっても問い合わせや確認は禁止です。MR Description とコメントから自律的に判断してください
5. 作業対象が特定できない場合でも、まずリポジトリ・ブランチ・MR本文・コメントを読み、必要最小限の実装案を自律的に決定してください
